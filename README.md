# Battle Brick - Fixed Local Version

This is a cleaned and runnable version of Battle Brick.

## What was fixed

- Rebuilt the app as a single-page game to avoid losing the Socket.IO connection during page redirects.
- Added proper 1v1 matchmaking by time control.
- Added isolated game rooms, so multiple games can run at the same time.
- Added server-side validation for row, column, value, active brick, filled brick, game status, and turn ownership.
- Moved timeout authority to the server.
- Added stable timers for `1+0`, `3+0`, `5+0`, and `10+0`.
- Added reconnect grace period after refresh or short disconnection.
- Added invalid-equation explanation and highlighted losing bricks.
- Removed the broken `timerOpponent`, client draw-check, and old redirect-based flow.
- Removed dependency on `noisejs`; the wall is generated by built-in server logic.

## Requirements

- Node.js 18 or newer
- npm

## Run locally

```bash
npm install
npm start
```

Then open:

```text
http://localhost:3000
```

To test locally, open the game in two browser tabs or two different browsers.

## Run on another PC in the same network

1. Start the server on your main PC:

```bash
npm start
```

2. Find your local IP address.

On Windows:

```bash
ipconfig
```

Look for the IPv4 address, for example:

```text
192.168.1.25
```

3. Open this from another device in the same Wi-Fi/network:

```text
http://192.168.1.25:3000
```

If it does not open, allow Node.js through Windows Firewall.

## Change port

```bash
PORT=4000 npm start
```

On Windows PowerShell:

```powershell
$env:PORT=4000; npm start
```

## Current game rule

A player chooses an empty brick and places one digit from 0 to 9. When a move completes a neighboring triple, the game checks whether the first two bricks sum to the third brick. If the completed equation is wrong, the player who made the move loses.
